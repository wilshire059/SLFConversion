# Animation Selection Guide

How to pick and validate animations for custom enemies from the Elden Ring catalog.

## Data Sources

### 1. Swing Direction Catalog (`swing_catalog.yaml`)
- **7,162 animations** across 288 characters
- Per-animation: direction, hit count, speed class, range, arc type, wind-up time, recovery time
- L_Hand behavior (guard_pose, active_strike, casting, passive)
- Generated by: `swing_partial/enhanced_analysis.py`
- Uses: R_Hand bone trajectory through parent chain to classify swing direction

### 2. Root Motion Catalog (`root_movement_results.txt`)
- **833 animations** with 2.5+ units of actual body displacement
- Uses: `reference_frame_samples` from HKX animation container (NOT bone translations)
- Generated by: `swing_partial/find_root_movement.py`
- Categories: BACKSTEP, LUNGE_FWD, SIDESTEP, DIAGONAL, BACK_THEN_LUNGE

### 3. Movement Catalog (`movement_catalog.yaml`)
- 1,411 animations tracked by stance center (avg of L_Foot + R_Foot)
- **WARNING**: This measures limb spread, NOT actual body translation. Use root_movement_results.txt instead for real displacement.

## Critical: Where Root Motion Lives

Elden Ring animations store root motion in THREE separate places:

| Location | What it contains | Useful? |
|----------|-----------------|---------|
| `Master` bone translation | Small oscillation (< 0.3 units) | NO — too small |
| `Root` bone translation | Forward/back weight shift (< 0.6 units) | NO — just weight shift |
| **`reference_frame_samples`** | **Actual game root motion (0-12+ units)** | **YES — this is the real data** |

`extract_animations.py` reads `reference_frame_samples` via `animation_container.get_reference_frame_samples()` and bakes it into the Blender object's `location` fcurve. This is why animations show movement in Blender but the bone data looks stationary.

### How to verify in Blender
Open a .blend file and check the **object-level** location fcurves (not pose bone fcurves):
- `location.X` = lateral movement (sidestep)
- `location.Y` = forward/backward movement (lunge/backstep)
- `location.Z` = vertical (jump)

Net displacement > 2.5 = real movement visible to the player.

## Animation Selection Process

### Step 1: Define Enemy Character
- Role: regular, elite, mini-boss, boss
- Personality: feral, disciplined, heavy, agile
- Weapon type: sword, dual-wield, fists, staff
- Has shield? (affects guard animation needs)

### Step 2: Pick Attacks (from swing_catalog.yaml)
For each attack slot, search by:
- **Direction**: Alternate R2L → L2R → overhead → diagonal → thrust for natural combos
- **Speed class**: Match enemy personality (feral = fast/medium, heavy = slow/very_slow)
- **Hit count**: 1 for singles, 2 for combo attacks
- **Source**: Use different ER sources per attack for forensic distinctiveness

**Combo flow rules:**
- Never use same direction twice in a row
- Alternate fast → slow or vice versa for rhythm variety
- First attack should be fast (opener), last should be slow (punishable)
- Heavy attack should have long wind-up (telegraphed)

### Step 3: Pick Dodges (from root_movement_results.txt)
Filter by:
- `direction = BACKSTEP` for backward dodge
- `anim_type = hit_react` for stumble-back (feral enemies)
- `anim_type = attack` for backstep-attack combos (smart enemies)
- `anim_type = dodge` for clean dodges (disciplined enemies)
- Duration < 3s for responsive dodges

**WARNING**: Many animations classified as "dodge" (5000-5999 range) are just pivots with no actual displacement. Always verify `reference_frame_samples` shows 2.5+ displacement.

### Step 4: Locomotion (always c3100)
All humanoid enemies share:
- Idle: c3100 `a000_000020`
- Walk: c3100 `a000_002000`
- Run: c3100 `a000_002100`

### Step 5: Core Animations (always c3100)
- Hit reacts: c3100 `a000_006001`, `a000_006002`
- Deaths: c3100 `a000_011010`, `a000_011011`, `a000_011012`
- Guard: c3100 `a000_007010`

### Step 6: Visual Validation
1. Extract picks via `extract_animations.py` → Blender
2. Build combo preview (NLA stitched sequence)
3. Open in Blender, press Space to play
4. Check for:
   - Pivoting without movement (bad dodge)
   - Sidestep that's actually a spin attack
   - Hit react that's actually falling down
   - Quadruped motion on humanoid rig (incompatible skeleton)
5. Iterate: swap bad picks, re-preview

### Step 7: Import to UE5
Update `animation_config.json` and run `SetupBatchEnemyCommandlet`.

## Common Pitfalls

### "Dodge" animations with no movement
Elden Ring 5000-5003 dodge anims are just upper-body pivots. The game engine applies displacement at runtime. Use 5010+ or attack/hit_react anims with root motion instead.

### Foot/stance analysis is misleading
Tracking L_Foot + R_Foot average position measures limb spread, not body translation. A spin attack shows as "sidestep" because feet spread apart. Always use `reference_frame_samples` for movement detection.

### Bone-level Root translation vs reference_frame_samples
The Root bone Y translation maxes out at ~0.5 units even for animations that visibly move 8+ units. The actual displacement is in `reference_frame_samples`, a separate HKX data channel.

### Quadruped animations on humanoid rigs
Some ER characters (Carian Knight dodges, some Grafted Scion anims) produce quadruped-looking motion when retargeted to a standard humanoid skeleton. Always preview before committing.

## V2 Pipeline: Direct Import + Forensic Post-Processing

The streamlined pipeline (v2) skips ARP retarget entirely. Cleaner results, fewer distortion issues, stronger forensic profile.

### Quick Reference (full enemy in ~15 minutes)
```bash
# 1. Pick attacks from catalogs, extract into rigged.blend via Blender
#    (use extract_animations.py import_animation_to_action)

# 2. User reviews rigged.blend in Action Editor, approves animations

# 3. Pre-forensic: strip translations, weapon fcurves, mirror lower body
blender --background --python pre_forensic_processing.py -- --enemy <name>

# 4. Full forensic: 13 transforms + rotation-space noise
blender --background --python apply_forensic_transforms.py -- --enemy <name>

# 5. Auto-generate hitbox timing from hand velocity
blender --background --python generate_hitbox_timing.py -- --enemy <name>

# 6. Export animation-only FBX (armature only, NO mesh)

# 7. Import to UE5
UnrealEditor-Cmd.exe SLFConversion.uproject -run=SetupBatchEnemy -name=<name>
```

### Forensic Safety: 9/10
- 13 transforms (offset, phase shift, noise, low-pass, time warp, speed, resample, decimation, prune, rest pose, bone rename)
- Pre-forensic strip (translations zeroed, weapon/hand/finger fcurves deleted — irrecoverable)
- Lower-body L/R mirror on 50% of animations
- Rotation-space noise (2° max) — changes motion direction, not just magnitude
- Multi-source mixing (5+ ER enemies per character)

### Critical Rules
- **FBX must be animation-only** (armature, no mesh) for UE5 Interchange importer
- **Animations are 24fps** after forensic resample — hitbox timing must account for this
- **Dodges need root motion enabled** (`bForceRootLock=false, bEnableRootMotion=true`)
- **Use different ER sources per enemy** for forensic mixing
